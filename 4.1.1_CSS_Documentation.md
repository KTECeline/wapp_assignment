# 4.1.1 CSS (Web Page Styling)

## Overview
The De Pastry Lab web application implements a modern, responsive design using three complementary CSS approaches: **External CSS** (Tailwind CSS), **Internal CSS** (style tags in React components), and **Inline CSS** (className utilities). The application uses a sophisticated styling architecture that combines a utility-first CSS framework with custom CSS rules for enhanced control and customization.

---

## 1. EXTERNAL CSS

### 1.1 Tailwind CSS Configuration
**File:** `Front/tailwind.config.js`

Tailwind CSS is the primary CSS framework used, configured with custom extensions for the application's design system.

**Code Snippet:**
```javascript
/** @type {import('tailwindcss').Config} */
module.exports = {
  content: ["./src/**/*.{js,jsx,ts,tsx}"],
  darkMode: 'class',
  theme: {
    extend: {
      fontFamily: {
        inter: ["Inter", "sans-serif"],               
        ibarra: ["Ibarra Real Nova", "serif"],         
      },
      colors: {
        pastry: {
          bg: '#FFF6F0',
          pink: '#FADADD',
          pink2: '#FCE2E0',
          gold: '#F4C27F',
        }
      }
    },
  },
  plugins: [
    require('tailwind-scrollbar-hide')
  ],
}
```

**Explanation:**
- **Content Configuration:** Scans all JSX/TSX files in the `src` directory to detect Tailwind classes
- **Dark Mode:** Supports dark mode using the `class` strategy
- **Font Extensions:** Defines two custom font families:
  - `font-inter`: Used for body text and UI elements
  - `font-ibarra`: Used for headings and titles
- **Color Palette:** Adds a `pastry` color scheme with 4 custom colors representing the bakery theme (#FFF6F0 for background, #FADADD for pink accents)
- **Plugins:** Includes `tailwind-scrollbar-hide` for custom scrollbar styling across the application

## 2. INTERNAL CSS (Style Tags in Components)

Internal CSS is embedded directly within React/TSX components using `<style>` tags with template literals. This approach allows for dynamic styling based on component state.

### 2.1 Scrollbar Customization
**File:** `Front/src/pages/RgUserCol.tsx` (Lines 223-233)

**Code Snippet:**
```tsx
<style>
    {`
    .no-scrollbar::-webkit-scrollbar {
        display: none;
    }

    .no-scrollbar {
        -ms-overflow-style: none;  
        scrollbar-width: none;  
    }
    `}
</style>
```

**Explanation:**
- **Scrollbar Hiding:** This CSS targets three browser scrollbar implementations:
  - `::-webkit-scrollbar` for Chrome, Safari, and Edge browsers (sets display: none)
  - `-ms-overflow-style: none` for older Internet Explorer/Edge versions
  - `scrollbar-width: none` for Firefox
- **Use Case:** Applied to containers that need to hide scrollbars while maintaining scroll functionality
- **Cross-Browser Compatibility:** Ensures consistent appearance across all major browsers

### 2.2 Custom Scrollbar Styling
**File:** `Front/src/components/PostForm.tsx` (Lines 125-145)

**Code Snippet:**
```tsx
<style>

        .scrol::-webkit-scrollbar {

        .scrol::-webkit-scrollbar-track {

        }

        .scrol::-webkit-scrollbar-thumb {
            background: rgba(0, 0, 0, 0.2);

        }

        .scrol::-webkit-scrollbar-thumb:active {

```

**Explanation:**
- **Scrollbar Size:** Sets both width and height to 5px for a thin, elegant scrollbar
- **Track Styling:** Semi-transparent gray background (rgba(188, 188, 188, 0.2)) with rounded corners
- **Thumb Styling:** Semi-transparent dark background with smooth transitions
- **Cursor Feedback:** Changes cursor to "grab" when hovering over the scrollbar, and "grabbing" when dragging
- **Animation:** Smooth 0.3s transition on hover for visual feedback

## 3. INLINE CSS (Tailwind Classes in JSX)

Inline styles are primarily implemented through Tailwind's utility-first class names directly in JSX elements. This approach provides rapid UI development with consistent design tokens.

### 3.1 Navigation and Button Styling
**File:** `Front/src/pages/RgUserCourse.tsx` (Lines 856-862)

**Code Snippet:**
```tsx
<button className="cursor-pointer uppercase hover:text-[#DA1A32] transition-all duration-[600ms]"
    onClick={() => navigate(`/RgUserLearn`)}>
    Learn
</button>
```

**Explanation:**
- **Cursor Pointer:** Changes cursor to hand on hover
- **Text Transform:** `uppercase` converts text to uppercase
- **Hover Effects:** `hover:text-[#DA1A32]` changes text color to the brand red (#DA1A32) on hover
- **Transition Animation:** `transition-all duration-[600ms]` creates a 600ms smooth animation for all property changes
- **Tailwind Arbitrary Values:** `[#DA1A32]` and `[600ms]` use square brackets for custom values not in the default Tailwind palette

### 3.2 Hero Section with Gradient and Backdrop Blur
**File:** `Front/src/pages/LandingPage.tsx` (Lines 210-225)

**Code Snippet:**
```tsx
<div className="w-full h-[500px] relative">
    <img src="/images/Announcement2.jpg" alt="announcement" className="w-full h-[500px] object-cover z-0" />
    <div className="absolute top-0 left-0 w-full h-[500px] bg-[#fefefe]/20 backdrop-blur-[12px] z-10" />
    <div className="absolute top-0 left-0 w-full h-[500px] z-20">
        <div className="absolute top-[120px] left-0 w-full text-white pl-[122px]">
```

**Explanation:**
- **Full Width Container:** `w-full h-[500px] relative` creates a responsive container
- **Image Positioning:** `object-cover` ensures image fills the container without distortion
- **Layered Effects:**
  - **Layer 1 (z-0):** Background image
  - **Layer 2 (z-10):** Semi-transparent overlay with `bg-[#fefefe]/20` (20% opacity) and `backdrop-blur-[12px]` (12px blur effect)
  - **Layer 3 (z-20):** Content overlay
- **Opacity and Blur:** `bg-[#fefefe]/20` applies opacity modifier, `backdrop-blur-[12px]` creates glass-morphism effect
- **Positioning:** Absolute positioning with z-index for layering control

### 3.4 Responsive Tabs with Animation
**File:** `Front/src/pages/RgUserCol.tsx` (Lines 237-250)

**Code Snippet:**
```tsx
<div className="relative flex flex-row font-ibarra text-[22px] font-medium mb-[36px] gap-[16px] rounded-full bg-[#F8F5F0] p-2">
    {tabs.map((tab) => (
```

- **Tab Button:** `rounded-full transition-all duration-300` - pill-shaped with smooth transitions
- **Active State Logic:** Conditional classname checks if tab is active:
  - Active: `text-[#DA1A32]` (brand red)
  - Inactive: `text-black`
- **Animation:** `transition-colors duration-300` smoothly animates color changes over 300ms
- **Z-Index:** Text uses `z-10` to appear above any background effects

## 4. EXTERNAL CSS LINKS (Google Fonts)
**File:** `Front/public/index.html` (Lines 8-19)

**Code Snippet:**
```html
<head>
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
  <link
    href="https://fonts.googleapis.com/css2?family=Inter:ital,opsz,wght@0,14..32,100..900;1,14..32,100..900&display=swap"
    rel="stylesheet">

  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
  <link
    href="https://fonts.googleapis.com/css2?family=Ibarra+Real+Nova:ital,wght@0,400..700;1,400..700&family=Inter:ital,opsz,wght@0,14..32,100..900;1,14..32,100..900&display=swap"
    rel="stylesheet">
</head>
```

**Explanation:**
- **Preconnect Links:** `rel="preconnect"` establishes early connections to Google Fonts servers, improving font loading performance
- **Inter Font:** Versatile sans-serif font for body text and UI elements
  - Weight range: 100-900
  - Supports italic variants
  - Optical sizing for optimized rendering across font sizes
- **Ibarra Real Nova Font:** Elegant serif font for headings and titles
  - Weight range: 400-700
  - Supports italic variants
- **Display Swap:** `display=swap` ensures text remains visible during font loading (Critical for web vitals)

## 5. CSS DESIGN SYSTEM SUMMARY

### Color Palette
| Variable | Hex Code | Usage |
|----------|----------|-------|
| `--bg` | #FFFFFF | Main background color |
| `--text` | #3D3D3D | Primary text color |
| `--accent` | #D9433B | Brand red for interactive elements |
| `--accent-dark` | #B13A33 | Darker red for hover/pressed states |
| `--border` | #F2E6E0 | Subtle borders |
| `--surface` | #FFF8F2 | Light tinted surface backgrounds |

### Typography System
- **Display/Headings:** Ibarra Real Nova (serif) - conveys elegance and bakery theme
- **Body/UI Text:** Inter (sans-serif) - modern, readable, accessibility-focused
- **Font Sizes:** Consistent 2px increments (12px, 14px, 16px, 18px, etc.)

### Spacing System
- Uses Tailwind's spacing scale (0, 2px, 4px, 6px, 8px, 12px, 16px, 24px, 32px, 36px, etc.)
- Arbitrary values with square brackets `[value]` for custom spacing

### Interactive Effects
- Hover transitions: 150ms to 600ms depending on context
- Lift effects on buttons: `translateY(-1px)`
- Color transitions: Smooth interpolation between states
- Cursor feedback: Changes to indicate interactivity

## 6. KEY CSS FEATURES

### 6.1 Responsive Design
- **Mobile-First Approach:** Tailwind utilities allow easy responsive design
- **Container Queries:** Fixed widths (1090px) for desktop with responsive adjustments
- **Flexible Layouts:** Flexbox and grid for adaptable layouts

### 6.2 Performance Optimizations
- **Critical CSS:** Tailwind purges unused styles in production
- **Font Optimization:** Preconnect links reduce font loading latency
- **Lazy Loading:** Images and videos loaded on demand
- **CSS-in-JS:** Only applied CSS is shipped to browser

### 6.3 Accessibility
- **Color Contrast:** Brand colors meet WCAG AA standards
- **Focus States:** Interactive elements have visible focus indicators
- **Motion Preferences:** Respects `prefers-reduced-motion` setting
- **Semantic HTML:** Proper heading hierarchy and landmark regions

---

## CONCLUSION

The De Pastry Lab application implements a sophisticated three-tier CSS architecture:

1. **External CSS (Tailwind):** Provides utility-first foundation with custom configuration for fonts and colors
2. **Internal CSS (Style Tags):** Handles dynamic styling and browser-specific customizations (scrollbars, webkit features)
3. **Inline CSS (Classes):** Rapid UI development through semantic utility classes

This combination delivers a performant, maintainable, and accessible web application with a cohesive design language that reflects the bakery theme and modern web standards.
